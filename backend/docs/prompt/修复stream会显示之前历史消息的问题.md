## 问题描述

当我现在 `POST /planner/plan/stream` 时 (具体的前后端交互接口详见 @backend/docs/api/前后端-接口文档.md ), 通过 apifox 预览时, 会展现出 @backend/docs/bug-desc/stream输出问题.png 中的结果, 简单来说就是: 先按照预期的逐 token 返回本次的回答, 但是随后又会直接再返回本次的完整回答, 然后还可能再返回上一轮对话的完整回答 (甚至是历史记录中的某几条完整回答), 然后才收到 `[DONE]` 信号. 但是调用 `GET /planner/history` 时 (也就是前端刷新页面之后), 则能够正确的返回无重复无遗漏顺序正确的历史记录.

我认为问题应该是出在后端 (毕竟通过 apifox 这样的 api test tool 调试都会出现这样的问题, 并不是只有前端实际调用时出现这个问题). 请你仔细阅读并深入理解当前后端 @backend/ 的架构与实现, 分析是什么导致了这个 bug 的产生, 随后规划一个修复该 bug 的方案.
如果有必要, 你可以 `git checkout` 到 `feat/frontend`, 在 frontend/ 目录下阅读并理解前端的实现, 不过所有对后端 bug 的修复规划和实施, 都要在当前的 `feat/backend` 分支上进行.

现在请你按照我的要求, 开始规划修复该 bug 的方案.
